Graph Layout Engine

How should nodes position themselves? This is the foundation of the Obsidian-like feel.

A

Force-Directed (like Obsidian)

Nodes repel each other, edges act as springs pulling connected nodes together. Produces organic, self-organizing clusters. This is exactly what Obsidian uses.

TECH: D3-force in 2D canvas or Three.js
Nodes float and settle naturally. Highly-connected modules cluster together. Isolated nodes drift to the edges. Real-time physics simulation.
B

Force-Directed 3D (cosmic upgrade)

Same physics simulation but in 3D space using Three.js + OrbitControls (which Atlas already has). Nodes become spheres in space connected by glowing lines. More immersive, keeps the cosmic aesthetic.

TECH: Three.js + d3-force-3d
Extends what Atlas already does. Reuses the existing Three.js setup, OrbitControls, raycaster, starfield. Adds edge lines + force simulation. Depth adds visual separation between clusters.
C

2D Canvas (lightweight)

Flat 2D graph rendered on HTML5 Canvas with pan/zoom. Simpler, faster, easier to add labels and overlays. Less immersive but more practical for daily use.

TECH: Canvas 2D + custom force sim or sigma.js
Fast rendering, crisp text, no WebGL required. But loses the cosmic 3D feel that Atlas currently has.
MY RECOMMENDATION
B — Force-Directed 3D. Atlas already has the Three.js engine, OrbitControls, raycaster, and starfield. Adding d3-force-3d gives us Obsidian's organic clustering in 3D space. We keep the cosmic immersion while gaining the relationship visibility you want. The edge lines between nodes will look like constellation links.
