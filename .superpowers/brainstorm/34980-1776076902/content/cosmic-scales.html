The Cosmic Scale Metaphor

How should the universe map to your codebase? Each scale level needs a visual identity.

A

Universe → Galaxy → Star System → Planet

4 zoom levels. Each click zooms into the next scale with a cinematic fly-through animation.

🌌
Universe
= Project
All top-level folders as galaxies floating in space
🌀
Galaxy
= Folder/Module
Spiral of files, sub-folders as arms. Click to enter.
☀️
Star System
= File
Central star (file) with orbiting planets (classes/functions)
🪐
Planet
= Class/Function
Methods as moons. Code viewer on click.
Deep metaphor. Most immersive. Animation-heavy. Complex to build but unforgettable UX.
B

Galaxy → Clusters → Stars

3 zoom levels. Simpler, flatter. Project is one galaxy, folders are clusters, files are stars.

🌀
Galaxy
= Project
Overview with all clusters visible
✨
Cluster
= Folder
Group of stars, force-directed within cluster
⭐
Star
= File
Click to see symbols in panel
Simpler to build. Fewer animation transitions. Obsidian-like but with clustering.
C

Hybrid: Universe view + Explode on click

Start with the current view (modules as nodes). Click a node → it EXPLODES into its internal structure with a cinematic animation. Like zooming from Google Maps satellite to street view.

●
root (collapsed)
→ click →
●
cli.py
●
server.py
●
db.py
●
analyzer.py
●
config.py
files inside root, sized by complexity, connected by imports
Progressive disclosure. Starts simple. Explodes on demand. Each level can explode further (file → functions). Most practical, still cinematic.
MY RECOMMENDATION
A — Full 4-level cosmic hierarchy. This is the signature experience that makes CopyClip unforgettable. The fly-through animation from universe → galaxy → star system → planet gives developers a visceral sense of scale and structure. It's the hardest to build but it's the experience that solves the cognitive debt problem — you can FEEL how big your project is and navigate it intuitively. Options B and C are compromises. CopyClip's vision deserves option A.
